Plugin Finder

Plugin Finder allows you to discover, install, update, and manage Mod Organizer 2 plugins from a curated directory of community plugins.

Features

  • Plugin Directory: Browse a curated list of MO2 plugins
  • Search & Filter: Find plugins by name, filter by status
  • One-Click Install: Install plugins directly without manual downloads
  • Update Detection: See which installed plugins have updates available
  • Version Tracking: View installed and latest versions for each plugin

Usage

Browsing Plugins

The main list shows all available plugins with their status, version information, author, and description. Each plugin entry includes links to GitHub, NexusMods (if available), and documentation.

Filtering

Use the filter options to narrow down the plugin list:

  • Search: Type to filter plugins by name
  • Installed: Show only installed plugins
  • Updates: Show only plugins with available updates
  • Supported: Show only plugins that support your MO2 version
  • Working: Show only plugins confirmed working with your MO2 version

Sorting

Sort the plugin list by name (ascending or descending) using the sort dropdown.

Installing Plugins

  1. Find the plugin you want to install
  2. Click the Install button on the plugin entry
  3. Restart Mod Organizer when prompted

Updating Plugins

Plugins with available updates show an update indicator. Click the Update button to download and install the latest version. Restart Mod Organizer to apply the update.

Uninstalling Plugins

Click the Uninstall button on an installed plugin to remove it. Note: Plugin Finder cannot uninstall itself.

Troubleshooting

A plugin installed through Plugin Finder does not work.

Plugins listed on Plugin Finder are maintained by their separate authors. If you have issues with a plugin, please check the documentation, GitHub and/or Nexus pages for the plugin.

A plugin listed in Plugin Finder is out of date.

Plugins listed on Plugin Finder are maintained by their separate authors. Please contact the author of the plugin and request they update their data for Plugin Finder.

A plugin I installed manually isn't listed as the correct version.

Plugin Finder attempts to detect the version based on the contents of the plugin, but this may not always be successful. When this fails, a default version number is given to the plugin. To remedy this, install or update the plugin through Plugin Finder.

Updating or uninstalling a specific plugin always fails.

Even if a message is logged in Mod Organizer that a file could not be moved/deleted, Plugin Finder will still attempt to move/delete those files when Mod Organizer restarts.

Sometimes current plugin files are locked or marked as read-only. To resolve this issue, manually delete the current plugin from Mod Organizer's plugins folder.

My plugin isn't listed in Plugin Finder.

If you have a plugin that you would like to add to Plugin Finder, you will need to create a manifest file and host it on the GitHub for your plugin.

A generator to help generate a manifest can be accessed here:
https://kezyma.github.io/?p=pluginfinder-generator

Once your manifest is on GitHub, open a pull request to add your plugin to the directory here:
https://github.com/Kezyma/ModOrganizer-Plugins/blob/main/directory/plugin_directory.json

When your pull request is approved, your plugin will appear in Plugin Finder.
