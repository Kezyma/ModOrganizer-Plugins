Root Builder

Root Builder allows you to manage files in the base game folder (not just the Data folder) through Mod Organizer 2. Keep the game folder clean, switch between mod configurations, and safely remove all root files when needed.

Features

  • Root Folder Management: Deploy files to the game's root directory
  • Multiple Modes: Copy, Link, USVFS, or Custom deployment methods
  • Auto Build: Automatically deploy and clean when launching games
  • Backup & Cache: Protect original game files
  • Exclusions: Skip specific files or folders
  • Import/Export: Share settings between instances

How It Works

Mods with a "Root" folder in their structure will have those files deployed to the game's root directory. Create a folder called "Root" in your mod and place files inside it matching the game folder structure.

Deployment Modes

Copy Mode

Copies all root files to the game folder. Simple and compatible but requires disk space for duplicates.

Link Mode

Creates file links instead of copying. Saves disk space but requires NTFS and may not work with all games.

USVFS Mode

Uses Mod Organizer's virtual file system. No actual file changes but may have compatibility issues with some games.

USVFS + Link Mode

Hybrid approach using USVFS for most files but links for executables (.exe) and libraries (.dll).

Custom Mode

Define your own rules for which files use which method. Set glob patterns and priorities for Copy, Link, and USVFS.

Tabs

Mode

Select the deployment method for your root files.

Custom Mode

Configure custom deployment rules with file patterns and priority settings for each method.

Settings

  • Cache: Cache base game files on first run for faster change detection
  • Backup: Create full backup of game files that may be modified
  • Autobuild: Automatically build and clear when launching applications
  • Redirect: Redirect exe files launched from mod folders to game folder
  • Installer: Enable automatic detection and installation of root mods
  • Hash: Use file hashing for accurate change detection (slower but more reliable)
  • Debug Level: Control logging verbosity

Exclusions

Define files and folders to exclude from Root Builder processing. Common exclusions include Saves, .ini files, and the Data folder.

Actions

  • Build: Deploy root files to the game folder
  • Sync: Update deployed files with current mod changes
  • Clear: Remove all deployed root files
  • Create/Delete Backup: Manage game file backups
  • Create/Delete Cache: Manage file cache

Export

Export settings to a file for backup or sharing. Import settings from a file or reset to defaults.

Troubleshooting

Mod Organizer warns me that mods with Root folders appear invalid or empty.

The Root folder isn't recognized by Mod Organizer as a valid folder and shows a warning. This can be safely ignored and will not stop the mod working with Root Builder.

Game files are being copied to the overwrite folder.

Root Builder may not detect game updates correctly for all games. To fix this:

  1. In the Actions tab, click Delete for both Backup and Cache
  2. Re-run your game update (Root Builder may have reverted it)
  3. In the Actions tab, click Create for both Backup and Cache

Running Build, Sync or Clear is slow or freezes Mod Organizer.

Root Builder can be slow with many files. To improve performance:

  • Remove unnecessary files from Root folders (documentation, source code)
  • Add large, unchanging game files to Exclusions
  • Disable Hash in Settings (less accurate but faster)
  • Disable Backup if game files won't be modified

Root Builder is not functioning correctly on Linux.

The autobuild functionality doesn't work on Linux. Run manual Build before launching applications and manual Clear after they close.
